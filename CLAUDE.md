# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

JPResume converts western-style markdown resumes to Japanese format (履歴書 rirekisho and 職務経歴書 shokumukeirekisho). Two parallel implementations exist: Swift (recommended) and Python (original).

## Build & Test Commands

### Swift (in `swift/` directory)

```bash
swift build                    # build
swift test                     # run all tests (15 tests, 3 suites)
swift run jpresume --help      # run CLI
swift run jpresume convert ../examples/resume.md --dry-run  # parse only
swift run jpresume convert ../examples/Kristopher_Baker_Resume.md --provider claude-cli --format both
xcodegen generate              # regenerate Xcode project from project.yml
```

### Python (in project root)

```bash
uv sync --extra dev            # install deps including test deps
uv run pytest -v               # run all tests (20 tests)
uv run pytest tests/test_parser.py::test_parse_name  # single test
uv run jpresume --help         # run CLI
uv run jpresume convert examples/resume.md --dry-run
```

## Architecture

Both implementations share the same pipeline: **Parse → Config → AI → Render**.

### Data Flow

1. **Parser** reads markdown into `WesternResume` (structured experience, education, skills, etc.)
2. **Config** loads `jpresume_config.yaml` (Japan-specific fields: kanji name, furigana, education dates, work history corrections) or prompts interactively, then saves for reuse
3. **AI** sends `WesternResume` + `JapanConfig` to an LLM, which returns `RirekishoData` / `ShokumukeirekishoData` as JSON. Results are cached to `.rirekisho_cache.json` / `.shokumukeirekisho_cache.json`
4. **Render** produces markdown (string templates) and PDF output

### AI Provider Abstraction

Protocol/ABC with `chat(system:user:) -> String`. Six implementations: Anthropic (URLSession + cache_control), OpenAI, OpenRouter (OpenAI-compatible), Ollama (local), Claude CLI (`claude -p`), Codex CLI (`codex exec`). Default provider is `ollama`. API providers use URLSession (Swift) or openai SDK (Python) directly — no Anthropic SDK dependency.

### Cross-Implementation Compatibility

Swift and Python share YAML config format and AI cache JSON format. CodingKeys in Swift models use `snake_case` to match Python's field names. Either implementation can read configs/caches generated by the other.

### PDF Rendering

- **Swift**: CoreGraphics directly (`CGContext` PDF context) with `NSFont(name: "HiraginoSans-W3")`. No font issues.
- **Python**: reportlab with STHeiti TTC font extraction via fontTools. fpdf2 used only for shokumukeirekisho. Historical note: fpdf2 has a font subsetting bug with CJK fonts across pages — this was the primary motivation for the Swift rewrite.

### Key Design Decisions

- The rirekisho PDF is a grid-form layout drawn with absolute coordinates, not HTML/template-to-PDF conversion
- System prompts explicitly instruct the AI to never fabricate dates or details
- Education entries support 卒業 (graduation) and 中途退学 (withdrawal) with configurable reasons
- The `work_japanese` config field stores additional work history not on the western resume (for the full timeline Japanese employers expect)
